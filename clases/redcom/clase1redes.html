1. ¿Qué es CISCO y su ecosistema de Redes?

Cisco Systems es el líder mundial indiscutible en el diseño, fabricación y venta de equipos de telecomunicaciones para redes [1]. El núcleo operativo de sus routers y switches tradicionales reside en el Cisco IOS (Internetwork Operating System), un sistema operativo de línea de comandos (CLI) altamente eficiente y estandarizado que rige el comportamiento de la infraestructura empresarial global.

"Dominar el CLI de Cisco IOS no solo capacita para administrar equipos de esta marca, sino que establece las bases lógicas universales para interactuar con casi cualquier sistema operativo de red del mercado."
— Adaptado de CCNA Study Guide, Cisco Press [1].
¿Por qué se utiliza Línea de Comandos (CLI)?

A diferencia de los entornos gráficos interactivos, la línea de comandos de Cisco IOS consume una cantidad mínima de ancho de banda y recursos de procesamiento. Esto permite que los ingenieros de red configuren y realicen diagnósticos de manera remota mediante protocolos seguros como SSH (Secure Shell) incluso en condiciones críticas de la red [2].

2. Modos de Operación de Cisco IOS

Para proteger la integridad del equipo, Cisco IOS utiliza un sistema jerárquico de accesos divididos en "modos". El prompt de la terminal cambia visualmente para indicar en qué nivel te encuentras:

Modo de Operación Prompt Visual Nivel de Acceso y Funcionalidad Comando de Entrada
Modo de Ejecución de Usuario Router> Solo permite comandos de visualización muy básicos y pruebas de ping/traceroute. No altera configuraciones. Modo inicial al conectarse.
Modo de Ejecución Privilegiado Router# Permite ver configuraciones detalladas, depurar y guardar cambios permanentes en el equipo. enable
Modo de Configuración Global Router(config)# Permite realizar cambios que afectan a todo el equipo de red (cambiar nombre, contraseñas, etc.). configure terminal
Modo de Configuración de Interfaz Router(config-if)# Configura puertos físicos específicos (FastEthernet, GigabitEthernet) del dispositivo. interface [nombre]

3. Comandos Esenciales de Configuración Inicial

Identificación del Equipo

Utiliza hostname [Nombre] en el modo global para dar una identidad lógica única al router o switch dentro de la topología de la red.

Seguridad Básica

Configura la contraseña de acceso privilegiado mediante enable secret [Contraseña]. Este comando cifra automáticamente la clave para mayor seguridad.

Encender Interfaces

Por defecto, los routers Cisco tienen sus puertos apagados administrativamente. Utiliza no shutdown dentro del modo de interfaz para activarlos.

Guardar Cambios

Los cambios que realizas van a la RAM volátil. Para guardarlos permanentemente en la NVRAM, utiliza el comando copy running-config startup-config.

Práctica de la Clase 2: Configuración de Router Cisco

Escribe la secuencia de comandos de Cisco IOS que daría solución a los siguientes requisitos lógicos empresariales.

Ejercicio 1: Securización Básica del Router

Indicaciones: Elabora la lista de comandos ordenada paso a paso que ejecutas a partir del prompt inicial Router> para:

  • Ingresar al modo global de configuración.
  • Asignar el nombre Sede-Central al equipo.
  • Establecer la clave encriptada de modo privilegiado como Cisco123.
  • Regresar de forma segura al inicio y guardar permanentemente los cambios.
Ejercicio 2: Activación de una Interfaz Local

Indicaciones: Un router Cisco posee la interfaz física apagada por defecto. Escribe la secuencia exacta de comandos en modo privilegiado (Router#) para:

  • Ingresar al modo de configuración de la interfaz FastEthernet 0/0.
  • Asignarle la dirección IP 192.168.1.1 con una máscara de red de 255.255.255.0.
  • Encender la interfaz físicamente para habilitar la transmisión de datos.
