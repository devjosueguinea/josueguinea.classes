1. Componentes de la Infraestructura

Para comprender la anatomía de una red, debemos clasificar sus elementos esenciales. Cada componente cumple un rol lógico y físico crítico para asegurar que los datos viajen de forma óptima, segura y sin pérdidas desde su origen hasta su destino.

Router (Enrutador)

Capa 3: Red

Es el cerebro de la interconexión. Su función principal es **unir redes completamente distintas** (como la LAN de tu empresa e Internet). Analiza las direcciones IP de los paquetes de datos y decide, mediante tablas de enrutamiento, cuál es el camino más rápido y eficiente para enviarlos.

Switch (Conmutador)

Capa 2: Enlace de Datos

Se encarga de **conectar los dispositivos locales dentro de una misma red** (LAN). A diferencia de los antiguos Hubs, el Switch es inteligente: lee las direcciones MAC físicas de las tarjetas de red y envía los datos exclusivamente al puerto del destinatario real, optimizando el ancho de banda.

Dispositivos Finales (Hosts)

Punto de Origen y Destino

Comprende todas las computadoras, servidores, impresoras de red, cámaras IP y teléfonos inteligentes. Son los elementos con los que interactúa el usuario; actúan como los **generadores o consumidores de la información**, iniciando solicitudes o respondiendo peticiones de servicios.

Punto de Acceso Inalámbrico (AP)

Extensión de Medios

Actúa como un puente de comunicación. Recibe la señal de red cableada del Switch y **la transforma en ondas de radiofrecuencia (Wi-Fi)**. Su propósito es permitir la movilidad de los dispositivos finales sin perder acceso a los recursos de la red local o corporativa.

Firewall (Cortafuegos)

Filtro de Seguridad de Red

Es la barrera de defensa perimetral. Inspecciona de forma exhaustiva todo el tráfico de datos entrante y saliente. Su rol es **bloquear accesos no autorizados y mitigar amenazas cibernéticas**, basándose en un conjunto estricto de reglas de seguridad corporativas previamente configuradas.

2. Métodos de Acceso a los Dispositivos

Antes de configurar el cerebro de un switch o router, necesitamos saber cómo conectarnos físicamente o lógicamente a él. Existen tres vías principales de acceso administrativo:

Puerto de Consola

Es una conexión física directa fuera de banda (Out-of-Band). Se usa un cable de consola especial conectado desde la PC al equipo de red. No requiere que el dispositivo tenga configuración de red previa; es ideal para configuraciones iniciales o rescates.

Secure Shell (SSH)

Método seguro de acceso remoto dentro de banda (In-Band). Cifra toda la comunicación (usuario, claves y comandos), evitando que atacantes intercepten los datos en tránsito por la red. Requiere una dirección IP configurada.

Telnet

Método remoto inseguro dentro de banda. A diferencia de SSH, transmite todo el tráfico en texto plano. Hoy en día se considera una mala práctica de seguridad y se encuentra en desuso en entornos corporativos reales.

3. Anatomía del CLI: Modos de Acceso y Comandos

Una vez conectados al sistema operativo (Cisco IOS), interactuamos con una interfaz de comando jerárquica. Despliega cada menú para explorar el propósito de cada nivel y los comandos fundamentales de red que puedes ejecutar en ellos:

Descripción: Es el modo de entrada por defecto al iniciar sesión. Está diseñado para tareas operativas extremadamente básicas de monitoreo y diagnóstico inicial. No permite modificar ningún parámetro físico o lógico del dispositivo.

Comando Tipo Función y Explicación Técnica
enable Transición Eleva tus privilegios en el sistema. Te desplaza directamente hacia el Modo de Ejecución Privilegiado.
ping [IP_Destino] Diagnóstico Envía paquetes de prueba ICMP Echo Request para comprobar la disponibilidad y latencia de un dispositivo remoto en la red.
traceroute [IP_Destino] Diagnóstico Rastrea la ruta física exacta (saltos de routers intermediarios) que toma un paquete de datos para alcanzar su destino.
show version Información Muestra información del hardware del router/switch, la versión exacta del sistema operativo Cisco IOS y el tiempo de actividad del equipo.
exit Sesión Cierra de forma segura la sesión activa de consola o terminal remota.

Descripción: También conocido como Enable Mode o modo de monitoreo avanzado. Otorga visibilidad completa de los archivos internos del sistema y el estado general de la memoria del router o switch.

Comando Tipo Función y Explicación Técnica
configure terminal Transición Permite dar el salto al Modo de Configuración Global para realizar cambios operacionales.
show running-config Información Visualiza la configuración que se está ejecutando actualmente en la memoria RAM volátil. Los cambios no guardados se listan aquí.
show startup-config Información Muestra el archivo de configuración almacenado de forma permanente en la memoria NVRAM (la configuración de inicio al encender el equipo).
copy running-config startup-config Almacenamiento Guarda y clona toda la configuración activa (RAM) dentro de la memoria permanente (NVRAM) para evitar pérdidas si el equipo se apaga.
reload Sistema Reinicia físicamente el hardware del equipo. Es útil para aplicar actualizaciones o restaurar valores de fábrica.
disable Sesión Reduce los privilegios de sesión, devolviendo al usuario al Modo de Ejecución de Usuario básico.

Descripción: Es el centro operacional real. Cualquier cambio que se ejecute en este entorno altera directamente los componentes físicos y lógicos de todo el dispositivo. Permite el ingreso a submodos de configuración específicos (puertos o líneas de comunicación).

Comando Tipo Función y Explicación Técnica
hostname [Nuevo_Nombre] Configuración Cambia el nombre de identificación lógica de la máquina. Este cambio altera de inmediato el texto mostrado en el prompt de comandos.
enable secret [Contraseña] Seguridad Establece una contraseña fuertemente encriptada con algoritmos hashing (MD5/SHA) para proteger la entrada al Modo Privilegiado.
interface [Nombre_Puerto] Transición Te traslada al submodo de configuración de una interfaz de red física o virtual para asignarle direccionamiento IP (Ej: interface FastEthernet 0/1).
line console 0 Transición Ingresa al submodo para configurar las políticas de acceso físico del puerto de consola (como contraseñas o tiempos de espera).
line vty 0 4 Transición Accede a las líneas de terminal virtuales para configurar accesos remotos seguros vía SSH o inseguros con Telnet.
exit Sesión Retrocede un nivel en la jerarquía del árbol de comandos (vuelve al Modo Privilegiado).
end Sesión Sale de cualquier submodo en el que te encuentres de forma directa, regresando al Modo Privilegiado (Router#) de un solo golpe.
Simulador Interactivo: Clasificador de Direcciones IP

Las direcciones IPv4 se dividen tradicionalmente en **5 clases (A, B, C, D y E)** de acuerdo con el valor decimal de su primer octeto.
