Introducción al Diseño de Sistemas Informáticos y Modelos SDLC

El Diseño de Sistemas Informáticos es la disciplina que define la arquitectura, módulos, interfaces y datos de un sistema de software para satisfacer requisitos específicos del negocio. En esta clase inicial, exploraremos cómo dar inicio a un proyecto tecnológico viable, las propiedades estructurales esenciales del código y las metodologías del ciclo de vida del desarrollo de software (SDLC).

Inicio de un Proyecto y Atributos Críticos del Sistema

El ciclo de vida de todo sistema informático comienza mucho antes de escribir código. Definir los cimientos arquitectónicos del software asegura que la solución pueda responder de forma eficiente a la demanda esperada de los usuarios y del negocio.

🚀 Inicio del Proyecto

Consiste en la identificación de la necesidad del negocio, la recopilación formal de requerimientos funcionales (lo que hace el sistema) y no funcionales (cómo lo hace, rendimiento, seguridad), el análisis de factibilidad y la delimitación estricta del alcance.

📈 Escalabilidad

Capacidad de una aplicación para responder óptimamente ante el incremento del volumen de trabajo sin perder rendimiento.

  • Vertical (Scale-up): Incrementar recursos (CPU, RAM) al servidor existente.
  • Horizontal (Scale-out): Añadir servidores en una red distribuida.

🛡️ Disponibilidad

Proporción de tiempo que un sistema está operativo, accesible y listo para realizar su función programada.

Se mide en porcentajes conocidos como "los nueves" (ej. 99.9%). Requiere mecanismos de redundancia y tolerancia a fallos.

Métricas de Diseño de Componentes: Acoplamiento y Cohesión

Para construir software que sea fácil de mantener, escalar y probar, los arquitectos se apoyan en dos conceptos métricos centrales de la Programación y el Diseño Orientado a Objetos:

La Regla de Oro del Diseño: Se debe buscar siempre el Bajo Acoplamiento (módulos comunicados mediante interfaces sencillas con mínimas dependencias mutuas) junto con una Alta Cohesión (cada clase o módulo se enfoca en hacer una única cosa extremadamente bien).

Modelos de Ciclo de Vida del Software (SDLC)

El Ciclo de Vida del Desarrollo de Software (SDLC) representa las metodologías y marcos de trabajo adoptados para estructurar, planificar y controlar el proceso de desarrollo de sistemas de información.

🌊 1. Modelo en Cascada (Waterfall)
Es el enfoque metodológico clásico y predictivo. El desarrollo fluye de forma secuencial hacia abajo a través de fases rígidas e inflexibles:
  1. Requisitos: Capturados y congelados al inicio.
  2. Diseño: Definición de arquitectura detallada.
  3. Implementación: Escritura del código del sistema.
  4. Pruebas: Control de calidad generalizado.
  5. Despliegue: Entrega final al cliente.
Ventajas: Estructura clara y de fácil gestión en proyectos con requisitos estáticos.
Desventajas: Inflexible al cambio; el cliente no ve software funcional hasta el final, elevando el riesgo de fallo.
⚡ 2. Modelo Ágil (Agile)
Enfoque incremental y adaptativo, enfocado en entregas tempranas de software funcional en ciclos fijos y cortos (sprints).

Pilares del Manifiesto Ágil:

  • Priorizar de forma primordial a los individuos e interacciones sobre procesos y herramientas.
  • Dar mayor importancia al software funcional por encima de la documentación exhaustiva.
  • Mantener colaboración activa con el cliente durante todo el proceso.
  • Respuesta oportuna ante el cambio sobre seguir un plan estricto.
Ventajas: Excelente adaptación al cambio y retroalimentación constante del cliente.
Desventajas: Dificultad para estimar tiempos y presupuestos finales exactos.
🌀 3. Modelo en Espiral (Spiral)
Creado por Barry Boehm, es un modelo guiado de forma sistemática por el Análisis y Gestión de Riesgos. Combina la naturaleza iterativa del prototipado con el control secuencial.

Cada iteración recorre obligatoriamente cuatro cuadrantes:

  1. Determinar objetivos: Identificar requisitos y restricciones.
  2. Analizar riesgos: Crear prototipos para mitigar problemas potenciales.
  3. Desarrollar y probar: Escribir el código y validar el software.
  4. Planificar: Evaluar resultados y preparar el siguiente ciclo.
Ventajas: Gestión de riesgos robusta, excelente para sistemas críticos (ej. militar, médico).
Desventajas: Complejo y costoso; depende críticamente de la experiencia en análisis de riesgos.
